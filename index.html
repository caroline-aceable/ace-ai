From #guild-ai

Questions we hear all the time

These came up again and again in #guild-ai. Real questions, answered by the team.

How do I get access to Claude or ChatGPT?
Use the /assist slash command in any Slack channel to submit an IT ticket. Say something like "Get set up on Claude Code." It usually goes to Eric and comes back same-day. If you're not sure whether you already have access, try logging into claude.ai with your Aceable email — if it works, you're in.
I see two accounts — "Aceable" and "Aceable Inc." Which one is right?
This one trips up a lot of people — and IT is your best resource here. There are two Claude instances and the right one for you depends on your role and how you were set up. If you're hitting usage limits unexpectedly, seeing missing features, or just not sure which account you're in — submit an IT ticket via /assist in Slack and ask them to confirm you're on the correct account. Don't assume — just ask. It takes two minutes and saves a lot of confusion.
Claude seems down or is throwing errors — is it just me?
Probably not just you. First check status.claude.com — Anthropic posts outages there in real time. If it's a known issue, just wait it out. If the status page says everything is fine but you're still having trouble, post in #guild-ai to see if others are affected too.
Claude Code terminal vs VS Code — which should I use?
Both work, but most Aceablers have landed on VS Code. It shows your files on the left, has a built-in terminal (press Ctrl+` to open it), and keeps everything in one window. Think of it as nesting dolls: VS Code is the outer shell, the terminal lives inside it, and Claude Code lives inside the terminal.

Start with VS Code. Use the terminal-only approach only if you have a specific reason to.
My sessions get slow or weird after a while. What's going on?
This is the context window filling up — Claude's short-term memory for a conversation. The longer you go, the more noise accumulates and the worse the answers get.

Two fixes:
• Type /clear to start completely fresh (wipes history, fast)
• Type /compact to summarize and free up space without losing all context

Best practice from the team: use /clear between different tasks rather than letting one session grow forever. One terminal tab per project works well.
How do I get access to Aceable's GitHub repos?
Submit an IT ticket via /assist in Slack asking for "Common Read Only role on aceable repo." Make sure your GitHub username follows the ace-firstname-lastinitial format so IT can find you. Once added, you'll get an email from GitHub — accept that invite before trying to clone anything.
I lost a whole day of work in v0. How do I avoid this?
Connect your v0 project to a GitHub repository from the start — that way every version is saved automatically. Also: watch out for the phrase "I need to write out the full file content" appearing in v0's chat. That's a warning sign it's about to rewrite something completely, which often breaks things. When you see it, stop and clarify exactly what you want changed before letting it proceed.
Is it safe to sign into Slack or Google through ChatGPT Agent mode?
Not without checking first. Agent mode creates a virtual computer that can log into services on your behalf — which means it could have access to things you didn't intend. Before connecting any work account through an AI agent, run it by Kevin or post in #guild-ai. The general rule: use designated group or service accounts, never your personal work email or any account linked to billing or password resets.
What VS Code extensions are actually worth it?
The team's crowd-sourced list: Claude Code (obviously), Peacock (color-codes different VS Code windows — lifesaver when you have multiple repos open), and PDF/XLSX viewer extensions so you can preview files without leaving VS Code. Beyond that, ask Claude: "What VS Code extensions would help me with [what you do]?" and it'll give you a personalized list.
How do I get a new AI tool approved?
Post in #guild-ai and tag Kevin. There's an approved tools list on Confluence — if something isn't on it, it needs a quick review before you use it on work projects. Most requests move fast. Don't sit on a free trial waiting — just ask early so you don't lose access.
Can Claude connect to Google Drive or Confluence?
Yes — Claude has connectors for both. For Confluence: go to Claude settings, find the Atlassian connector, and authorize it. Once connected, Claude can read and update Confluence pages directly. For Google Drive: the connector works but can be finicky — if it's not finding your files, try using "Add from Drive" to search directly rather than pasting a link. If you're in the wrong Claude account (Aceable Inc instead of Aceable), connectors won't work right either.
Can I break my computer by typing in the terminal?
No. The terminal is just a text box. If something looks wrong, close the window and open a fresh one — that fixes 90% of issues. If Claude does something unexpected, tell it: "undo that" or "go back to how it was." It's not fragile. The worst that usually happens is an error message, not actual damage.

The Aceable Vibe Coder Project Audit Prompt

Open your project folder in Claude Code, copy this entire prompt, paste it into Claude, and press Enter. Claude will audit your project for security issues and flag anything that needs attention.

# Aceable Vibe Coder Project Audit

You are auditing this project for security, legal compliance, and operational safety. We build educational software used by teenage students. Any mishandling of student data is a legal liability under COPPA and FERPA, not just a bug.

Start by detecting which platforms are in use:
- Full-stack app — package.json, API routes, or a web server
- n8n — workflow JSON exports or n8n references
- Google Apps Script — .gs files or appsscript.json
- Slack — Slack API calls, bot tokens, or webhook URLs
- Email — SMTP config, email API calls, or email-reading logic
- AI/LLM — calls to OpenAI, Anthropic, or any other model API

Only run sections relevant to the detected platforms. For every issue found: show the file and line, explain the risk, and give the fix. If a section is clean, say so explicitly.

---

## PART 1 — AUDIT

### STUDENT DATA & LEGAL COMPLIANCE — All platforms
This section is mandatory regardless of platform. Do not skip it.

- Does this project access, store, process, or transmit any data belonging to students under 18 — including names, emails, ages, course progress, scores, or any identifier that could be linked to a minor? If yes, flag every location.
- Is any student data being used in a development or test environment? Real student data must never leave production systems.
- Is any student data being sent to an external AI/LLM API as prompt content, context, or fine-tuning data? This is a potential FERPA/COPPA violation. Flag every instance.
- Is there a documented data retention policy for any student data stored by this project?
- Are there any fields that collect date of birth or age? If yes, is there logic to apply stricter handling for users under 18?
- Is student PII appearing in logs, error messages, analytics events, or third-party monitoring tools?

### SECRETS & ENV
- Are any API keys, secrets, or credentials hardcoded anywhere in the codebase?
- Are credentials or tokens appearing in any log output?
- Is .env and all variants in .gitignore?
- Are all secrets accessed via process.env and never passed as plaintext between files?

### AUTHENTICATION & AUTHORIZATION
- Are all protected routes covered by auth middleware — including API routes, not just UI pages?
- On every mutating endpoint (POST/PUT/PATCH/DELETE), is resource ownership verified server-side?
- Is any endpoint accepting a userId from the request body that should instead come from the session?
- Are admin-only routes gated by role check, not just login state?

### AI / LLM — if detected
- Is any student data included in prompts sent to an external model API? Flag every instance.
- Is user-supplied input passed directly into a prompt without sanitization?
- Is AI-generated output rendered in the UI as raw HTML without sanitization?
- Are model API keys stored securely in environment variables — never hardcoded or exposed to the client?
- Is the model being called from the client side (browser/frontend)? If yes, the API key is exposed to every user — this must be moved server-side immediately.

### GENERAL
- Are console.log or logger calls emitting PII, tokens, or passwords?
- Are there TODO or FIXME comments related to security, auth, or data handling?
- Are dependencies free of known high/critical vulnerabilities? Run npm audit or equivalent.
- Are inactive workflows, scripts, or services disabled or deleted?

---

## PART 2 — CLAUDE.md BOOTSTRAP

After completing the audit, check for a CLAUDE.md at the project root.
- If it does not exist, create it.
- If it exists, extend it with any rules below that are missing.
- Show me the full final file contents and wait for my approval before continuing.

The Non-Negotiable Rules

Even if you never write a line of code, these are the rules every Aceable vibe coder needs to know. If Claude ever suggests something that breaks these rules — stop, don't do it, and post in #guild-ai.

🚫

Never share student data with AI

Never include student names, emails, ages, scores, or any identifier in a prompt to Claude or any other AI. This is a COPPA and FERPA violation — not just a bug.

🚫

Never hardcode passwords or API keys

Never put passwords, API keys, or secrets directly into your code or files. They always go in environment variables. If you see one in code, it's a security incident.

🚫

Never connect personal email to automation

Your personal work email must never be connected to any automation, bot, or AI tool. Only use designated group or service accounts (support@, notifications@).

✅

Run the audit on every project

Before sharing or deploying anything, paste Kevin's audit prompt into Claude and let it run. It takes less than a minute and catches issues before they become problems.

✅

Use synthetic data for testing

When building or testing anything, use made-up data. Never use real student records — even in development or for a quick test.

✅

When in doubt, ask in #guild-ai

If Claude suggests something that feels off, or you're not sure if something is safe — stop and ask. Kevin, Peter, and the team are there to help.

Plain English

Jargon Decoder

Tech has a lot of words with extremely unhelpful names. Here's the plain English version.

Terminal also: CLI, command line

"It's just a text box"

A place where you type instructions and your computer follows them. Not hacking. Not scary. Just typing.

Prompt

"The instruction you give AI"

What you type to Claude. Better prompts = better results. Like giving directions — the more specific, the better.

Repository repo

"A folder with version history"

Like a Google Doc with track changes, but for any type of file. Lives on GitHub. "Clone the repo" = download the folder.

Git

"Save As, but on steroids"

A tool that tracks every change to your files over time. "Commit your changes" = save a snapshot of your work.

Directory

"A folder. That's it."

Developers call folders "directories" because they like sounding fancy. "Navigate to the directory" = go to the folder.

Context Window

"Claude's short-term memory"

How much text Claude can "remember" in one conversation. Long chats use it up. Start fresh sessions for new tasks.

MCP

"A plugin that connects Claude to other tools"

Model Context Protocol. Lets Claude Code plug into Slack, Google Drive, Jira, etc. Run /security-review before using any new one.

Deploy

"Put it on the internet"

Making something available for people to use, usually by hosting it online. Like publishing a blog post, but for any project.

Hallucination

"When AI makes stuff up"

When Claude confidently generates information that sounds correct but is wrong. Always review AI output for anything important.

Localhost

"Your computer pretending to be a website"

A way to test something on your own computer before sharing it. Only you can see it. Usually shows as localhost:3000 in your browser.

Pull Request PR

"Raise your hand to submit changes"

A proposal to merge your changes into a shared project. Like saying "I made some edits, can someone review them?"

CLAUDE.md

"Claude's memory file for your project"

A file you create that teaches Claude your preferences, style, and project context. It reads this every session so you don't have to repeat yourself.
